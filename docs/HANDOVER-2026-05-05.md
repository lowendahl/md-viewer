## Repo layout & gotchas

```
md-viewer/
  main.js              ← Electron main process. Auto-updater, IPC, single-instance,
                          open-file, open-url, mdviewer:// argv parsing.
  preload.js           ← contextBridge surface. Add new IPC methods here.
  index.html           ← Sidebar / topbar markup. #root-path is now an <input>.
  styles.css
  src/
    renderer.js        ← *** ACTIVE *** renderer entry point. All tree, file ops,
                          selection, history, find/replace, gallery live here.
  renderer.js          ← LEGACY. Do not edit. Builds ignore it.
  build.js / dev.js    ← esbuild bundle for src/renderer.js → dist/renderer.js
  CHANGELOG.md
  package.json         ← electron-builder config under "build". version is the
                          release tag.
  release/             ← electron-builder output (gitignored)
```

**Two `renderer.js` files.** All edits go to `src/renderer.js`. The
top-level `renderer.js` is dead weight kept around for archaeology.

**Bundling.** `node build.js` runs esbuild and writes `dist/renderer.js`
+ theme CSS. `npm run release` chains build → electron-builder → publish.

## How to cut a release

```powershell
cd C:\Users\plwendahl\repos\md-viewer

# 1. Bump version + write CHANGELOG entry.
#    Edit package.json "version" and CHANGELOG.md.

# 2. Verify the bundle compiles before tagging.
node build.js

# 3. Commit, tag, push.
git add -A
git commit -m "vX.Y.Z: <summary>"
git tag vX.Y.Z
git push origin main --tags

# 4. Publish. We use the gh CLI's lowendahl identity so the token has
#    write access to the public repo (the default identity is the EMU
#    account which can't see lowendahl/md-viewer).
$env:GH_TOKEN = (gh auth token --user lowendahl)
npm run release

# 5. Sanity-check.
gh release view vX.Y.Z --repo lowendahl/md-viewer `
  --json tagName,name,assets `
  -q '{tag:.tagName, name:.name, assets:[.assets[].name]}'
```

Expected assets per release: `latest.yml`, `MD-Viewer-X.Y.Z-x64.exe`,
`MD-Viewer-X.Y.Z-x64.exe.blockmap`, `MDViewer-X.Y.Z-portable.exe`.

If `latest.yml` is missing, the auto-updater will not see the release.

## Auto-updater architecture

- Reads `https://github.com/lowendahl/md-viewer/releases/latest/download/latest.yml`.
- Anonymous fetch — public repo, so no token needed on the client.
- `main.js` `wireAutoUpdater` runs on app ready; silent check fires after
  startup. `ipcMain.handle('updates:check')` is the manual entry point;
  it now translates SAX, YAML, 404, and network errors into friendly
  toast messages instead of leaking raw stack traces.
- **Don't move releases back to the EMU repo.** SAML SSO redirects
  `latest.yml` to an HTML login page; electron-updater feeds that to its
  YAML parser and crashes with `Attribute without value Line: 26 Column:
  76 Char: >`. The EMU repo `plwendahl_microsoft/md-viewer` still holds
  the v1.3.2 tag for archaeology only.

## `mdviewer://` protocol (1.3.6)

- Registered via `app.setAsDefaultProtocolClient('mdviewer')` in
  `main.js`, plus `package.json` → `build.protocols` so NSIS writes the
  registry keys on install.
- URL shape: `mdviewer://<absolute-path-to-md-file>`. Paths with spaces
  must be URL-encoded (`%20`).
- Windows: arrives as an argv element on second-instance; parsed by
  `extractMdPathFromArgv`. macOS: `app.on('open-url')`. Both feed into
  `pendingOpenPath` / the `'open-md-from-os'` IPC, which the renderer
  already handles.
- The path bar in the sidebar accepts `mdviewer://…` and `file://…` too,
  so users can paste either form.

## Known follow-ups / nice-to-haves

- **Recent paths in the path bar.** We track `pushRecent(folder)` but
  there's no dropdown UI to surface it. A small datalist on `#root-path`
  would be a 20-line job.
- **Forward-history truncation feedback.** Going back then loading a new
  root silently drops the forward stack. Could flash the Forward button
  to make this visible.
- **`mdviewer://` confirmation dialog.** Windows prompts the user the
  first time a protocol fires. We could pre-register on first launch to
  smooth this, but it's not blocking.
- **EMU repo cleanup.** `plwendahl_microsoft/md-viewer` should probably
  be archived with a README pointing at `lowendahl/md-viewer`. Not done
  yet because EMU archive UI requires SSO acrobatics.
- **Automated CI release.** Today releases are local-only (signtool +
  `gh auth token --user lowendahl`). Moving to GH Actions would need
  the signing cert as a secret — out of scope for now.
